Strategy Pattern

* * *

Sample 

interface PaymentStrategy {	def pay(amount) }      

class CreditCardStrategy implements PaymentStrategy  {
    def pay(amount) { println "$amount paid with credit card}
}

class PayPalStrategy implements PaymentStrategy {
    def pay(amount) { println "$amount paid using paypal"}
}

class Item { private String name, private int price }

class ShoppingCart {

    private List items = []

    def pay(strategy) {
        items.each {
            strategy.pay(it.price)
        }
    }

    def addItem(name, amount) {
        items.add(new Item(name: name, price: amount))
    }

}

def shoppingCart = new ShoppingCart()
shoppingCart.addItem("Sneaker", 100)

shoppingCart.pay(new CreditCardStrategy ())

Benefits

  • Polymorphie

    Alternative zur Unterklassenbildung

  • Code Recycling

    Das Verhalten wird durch die einzelnen Klassen contextunabhängig

  • Dynamisches Verhalten

    Änderung des Verhaltens zur Laufzeit

  • Vermeidung von fest kodierten Algorithmen

    Die Kohäsion des Contexts steigt

Disadvantages

  • Over-Engineering

    Es werden viele Klassen erstellt

  • Enge Kopplung zwischen Client und Strategieimplementierungen

    Der Client setzt die passende Strategie des Contexts und muss folglich die Implementierungen kennen
